# DuoPlan 开发日志

## 项目基准
- **技术栈**：React 18 + TypeScript + Vite + Tailwind CSS + Canvas 2D API + Yjs + Dexie.js
- **核心架构**：单页 B5 逻辑宽度 720px，纵向无限延伸，横向双页并排（1464px = 720×2 + 24 缝隙）
- **同步策略**：Yjs + y-webrtc P2P，单一共享 Y.Doc，notebooks[userId] 分区
- **主色调**：黑白灰 + 浅紫 #A78BFA
- **构建产物**：506KB JS (157KB gzip) + 20KB CSS
- **当前版本**：Session 17

### 技术决策（ADR-001）
- **决策**：Yjs 同步策略——单一共享 Y.Doc + notebooks[userId] 分区
- **原因**：双方各写各的笔记本条目，CRDT 保证无冲突；涂鸦写入对方条目

### 技术决策（ADR-002）
- **决策**：块级 Canvas 覆盖层 + 页面级 Canvas 双层架构
- **原因**：块级 Canvas（z-index 10）覆盖在块内容上方支持局部手写，页面级 Canvas（z-index 5）支持整页自由绘制
- **影响**：每个 BlockItem 挂载独立 HandwritingCanvas，页面还有一个全局 HandwritingCanvas

### 技术决策（ADR-003）
- **决策**：防误触——仅接受 pen 类型 PointerEvent 用于书写
- **原因**：平板触摸用于滚动，手写笔用于绘制，鼠标仅在涂鸦/绘制模式下可用
- **影响**：usePointerHandler 三分支过滤逻辑

## [2026-05-31] Session 11（当前）: 布局优化 — 侧边栏动效 + 无滚动条 + 缩放一致性

### 会话上下文
- **前置会话**：Session 10 涂鸦同步修复，Session 9 引入 absolute 定位侧边栏
- **遗留问题**：
  1. 侧边栏 absolute 覆盖方案不符合用户预期的推挤式交互
  2. CanvasScaler `overflow-auto` 导致画布外出现滚动条
  3. 房间码复制后因重渲染而刷新
  4. 缩放比例手感与浏览器不一致
- **本次目标**：恢复侧边栏推挤动效，消除滚动条，修复房间码稳定性，统一缩放手感

### 变更清单
| 文件 | 变更类型 | 说明 | 关联需求 |
|------|---------|------|---------|
| `src/App.tsx` | 修改 | 恢复 Sidebar + CanvasScaler flex 行布局（撤销 absolute） | #侧边栏动效 |
| `src/components/layout/CanvasScaler.tsx` | 重写 | 视觉尺寸容器（width=logicalWidth×scale）+ transform-origin:top left | #无滚动条 |
| `src/hooks/useCanvasScale.ts` | 修改 | 新增 containerHeight、zoomBy 加法→乘法 | #缩放一致性 |
| `src/components/room/CreateRoom.tsx` | 修改 | roomCode/userId 改为 useState 惰性初始化 | #房间码稳定性 |

### 技术决策（ADR-006）
- **决策**：CanvasScaler 采用视觉尺寸外层容器（width = logicalWidth × scale）
- **原因**：CSS `transform: scale()` 不改变元素布局尺寸，导致布局宽=1464px 产生溢出滚动条。视觉容器约束实际渲染宽度，无横向溢出
- **替代方案**：`overflow-x: hidden` + 裁剪（已否决，会裁剪居中后两侧内容）
- **影响**：transform-origin 从 `top center` 改为 `top left`，内部元素从容器左上角开始缩放

### 技术决策（ADR-007）
- **决策**：手动缩放从加法模型改为乘法模型（`prev * (1 + delta)`）
- **原因**：浏览器原生缩放和常见设计工具的缩放手感均为乘法，加法在多次缩放后手感漂移
- **影响**：delta 值相应调整（`-e.deltaY * 0.005`）

### 实现细节
- 侧边栏使用已有 `transition-all duration-200`，宽度变化时 CSS 自动插值
- ResizeObserver 在动画期间 ~60fps 回调，autoScale 连续更新
- 视觉容器 `mx-auto` 居中，双页缝隙对齐视口中轴

### 会话上下文

### 问题诊断（第二轮测试反馈）

| # | 问题 | 根因 | 修复 | 文件 |
|---|------|------|------|------|
| 1 | 笔迹偏移 | `getBoundingClientRect()` 返回 CSS transform 缩放后视觉坐标，Canvas 内部用逻辑坐标 | 坐标除以 `canvasScale` 转回逻辑坐标系 | usePointerHandler.ts, useUIStore.ts, App.tsx |
| 2 | 笔画不实时 | `handleStrokeMove` 忽略 `points` 参数，`currentStrokeRef` 从未更新 | 接收 points → 构建临时 Stroke → 存入 currentStrokeRef → redrawAll 渲染 | HandwritingCanvas.tsx |
| 3 | 连接离线 | `demos.yjs.dev` 被墙 + Heroku 免费层已关停 | 移除 y-websocket 中继，回到纯 WebRTC；新增 Nextcloud STUN 备选 | yjs.ts, useYjsSync.ts, constants.ts |

### 技术决策（ADR-010）
- **决策**：坐标修正——`(clientX - rect.left) / canvasScale`
- **原因**：CanvasScaler 使用 CSS `transform: scale()` 缩放，`getBoundingClientRect()` 返回视觉尺寸，但 Canvas 内部 ctx 坐标不受 CSS transform 影响，必须手动修正
- **影响**：`canvasScale` 存入 `useUIStore`，`usePointerHandler` 读取并应用逆变换

### 技术决策（ADR-011）
- **决策**：实时笔画——`handleStrokeMove` 接收 `Point[]` 并构临时 Stroke
- **原因**：`usePointerHandler.onStrokeMove` 回调已传递所有采样点，但 `HandwritingCanvas` 未使用；临时存入 `currentStrokeRef` 后 `redrawAll()` 即可实时渲染
- **影响**：用户画每一笔都能即时看到，不再等笔画结束

### 技术决策（ADR-012）
- **决策**：放弃 WebSocket 中继，回到纯 WebRTC P2P
- **原因**：y-websocket 依赖公开中继 `demos.yjs.dev`，在国内被墙；Heroku 免费层已于 2022 年关停
- **替代方案**：自建中继服务器（Fly.io 香港节点，免费），待用户确认后实施
- **影响**：同热点下 WebRTC 理论上可直连（仅需信令交换局域网 IP），但若 `signaling.yjs.dev` 也被墙则仍无法连接

## Session 15 — iPad 手写体验全面优化

参考 tldraw（37k star）的输入架构，系统性地修复了 iPad Apple Pencil 书写的四个问题。

### 完成事项

1. **上半页面无法起笔**（[PageCanvas.tsx](src/components/canvas/PageCanvas.tsx)）
   - **根因**：块容器 `z-index:6` > 页面 Canvas `z-index:5`，拦截了笔事件
   - **修复**：块容器设 `pointer-events: none`，每个 BlockItem 恢复 `pointer-events: auto`
   - 空区域笔触穿透到页面 Canvas，块自身保持可交互

2. **快速书写丢笔画：getCoalescedEvents**（[usePointerHandler.ts](src/hooks/usePointerHandler.ts)）
   - **根因**：iPad Safari 笔采样 120Hz 但只派发 60Hz 事件，丢掉一半采样点
   - **修复**：`onPointerMove` 调用 `e.nativeEvent.getCoalescedEvents()` 取出所有中间采样点
   - Canvas 坐标转换重构为接受 `clientX/clientY`，兼容原生 PointerEvent

3. **隔一笔丢一笔：pointerType 误判**（[usePointerHandler.ts](src/hooks/usePointerHandler.ts)）
   - **根因**：iPad Safari 快速书写时偶尔将 Apple Pencil `pointerType` 误报为 `touch`
   - **修复**：`shouldHandle` 放宽 → `isDrawing.current` 时不受 pointerType 变化影响，`touch + pressure > 0` 也视为笔

4. **pointerleave 不再截断笔画**（[usePointerHandler.ts](src/hooks/usePointerHandler.ts)）
   - 快速书写时 Apple Pencil 短暂离开悬停范围触发 `pointerleave`，之前调用 `onPointerUp` 截断笔画
   - 改为空操作，仅 `pointerup` 负责结束笔画

5. **新增 pointercancel 处理**（[usePointerHandler.ts](src/hooks/usePointerHandler.ts)）
   - 系统中断（来电/手势/手掌误触）时保存已有笔画数据后清理状态
   - 参考 tldraw 显式处理此事件

6. **iPad Safari 蓝色选中框禁用**（三处）
   - 全局 CSS：`canvas { user-select: none; -webkit-user-select: none; -webkit-touch-callout: none }`
   - 纸面 div：`userSelect: none; WebkitTouchCallout: none`
   - HandwritingCanvas inline style 同步加固

### iPad 手写输入保障体系（三层）

```
Layer 1: getCoalescedEvents()  → 补齐浏览器丢掉的采样点（120Hz→60Hz）
Layer 2: shouldHandle 放宽     → 容忍 pointerType 误判（pen→touch）
Layer 3: pointerleave 不截断   → 容忍短暂悬停丢失
         pointercancel 保存    → 系统中断不丢数据
```

- `src/hooks/usePointerHandler.ts` — getCoalescedEvents + shouldHandle 放宽 + pointerleave 空操作 + pointercancel
- `src/components/canvas/PageCanvas.tsx` — 块容器 pointer-events:none + 纸面 user-select 禁用
- `src/components/canvas/HandwritingCanvas.tsx` — user-select 禁用 inline
- `src/index.css` — canvas 全局禁用选中

## Session 16 — 双 Canvas 架构消除延迟 + 代码结构重构

### 1. 双 Canvas 架构（HandwritingCanvas 重写）

Session 15 的 getCoalescedEvents 修复了"丢点"问题，但延迟的根因仍在渲染管线：

```
旧架构（单 Canvas）每帧执行:
  clearRect() → renderStrokes(所有历史笔画) → renderStroke(当前笔画)
  O(n) 每帧，n=历史笔画数 → 笔画越多越卡
```

**新架构：双 Canvas 叠加**

| Canvas | z-index | 职责 | 何时渲染 |
|--------|---------|------|---------|
| 背景 Canvas | 5/10 | 已完成的笔画 | 笔画结束时渲染一次 |
| 前景 Canvas | 6/11 | 当前正在绘制的一笔 | 每帧 clear + render 1 条 |

```
新架构每帧执行:
  前景 cv.clearRect() → renderStroke(当前1笔)  ← O(1)!
  背景 cv：不动（笔画结束时才 renderStroke 一次）
```

**这是 tldraw / Excalidraw / GoodNotes 等主流绘图应用的标准做法。**

关键改动（[HandwritingCanvas.tsx](src/components/canvas/HandwritingCanvas.tsx)）：
- `canvasRef` → `bgCanvasRef` + `fgCanvasRef`
- `redrawAll()` → `redrawBackground()` + `redrawForeground()`
- 指针事件仅绑定在前景 Canvas，背景 `pointer-events: none`
- 笔画结束：先 `renderStroke` 到背景（即时反馈），再存入 store，最后清除前景
- 尺寸同步：两个 Canvas 同时应用 DPR 缩放

### 2. useNotebookStore 精简

每一个 mutation 都在重复相同的 pages-clone 模式：
```typescript
const pages = [...s.myNotebook.pages]
const page = { ...pages[pageIndex] }
// ... mutate page ...
pages[pageIndex] = page
```

**提取 `updatePage` 辅助函数：**
```typescript
function updatePage(pages, pageIndex, fn) {
  const updated = [...pages]
  updated[pageIndex] = fn({ ...updated[pageIndex] })
  return updated
}
```

所有 mutation 从 8 行缩到 4 行，可读性显著提升。修改文件：[useNotebookStore.ts](src/store/useNotebookStore.ts)
